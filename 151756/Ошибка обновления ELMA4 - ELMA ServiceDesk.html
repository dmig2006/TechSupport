Добавлен комментарий Гусаков Д. 18.07.2022 16:01:52
Проверьте, хватает ли свободного места на диске, куда система пытается создать резервную копию базы. Папку, в которую система создаёт бекап, можно посмотреть в файле конфигурации configuration.config (по умолчанию ..\ELMA3-Standart\UserConfig\configuration.config). В этом файле есть строка (по умолчанию 12): <main connectionStringName="MainDB" type="EleWise.ELMA.Extensions.MSSQL.MSSQLProvider, EleWise.ELMA.Extensions.MSSQL" backupEnabled="true" backupPath="C:\ELMA3_backup"/> Можно отключить создание бекапа при запуске элма-сервера. Для этого приведите данную строку к виду: <main connectionStringName="MainDB" type="EleWise.ELMA.Extensions.MSSQL.MSSQLProvider, EleWise.ELMA.Extensions.MSSQL" backupEnabled="false" backupPath=""/> Если выше все хорошо, то попробуйте следующее: Создайте резервную копию базы данных через программу СУБД https://www.elma-bpm.ru/KB/article-5793.html Создайте новую базу данных Восстановите резервную копию БД
